# Prompt #05 Design Truth Table

| Surface | Prior truth | Prompt #05 verdict | Prompt #05 move |
| --- | --- | --- | --- |
| Onboarding | Strong tone, weak ritual energy | Keep tone, compress authority, add live preview | Rebuilt as setup ritual + pulse preview |
| Top summary strip | Clear but passive | Needed real movement, not just metrics | Added delta-stage and action queue |
| Urgent stories | Right concept, weak execution | Too card-like and too quiet | Rebuilt as a distinct reel with countdown and trust |
| Fresh jobs list | Useful but gentle | Needed stronger ranking and action cues | Added delta badges, why-now cues, sharper card hierarchy |
| Job detail | Informative but isolated | Needed more outcome energy | Added trust check, application pulse, employer proof |
| Saved searches | Informative only | Not return-driving enough | Reframed as live, predictive cards |
| Alerts | Static cards | Needed clearer action tiers | Reframed as alert queue with modes |
| Story composer | Present but toy-like | Entry point felt too tucked away | Elevated to first-class launch actions and sheets |
| Employer composer | Correct idea, weak weight | Did not feel operational | Added verification-first sheet language |
| Trust explanation block | Conceptually useful | Too appendix-like | Absorbed into cards, detail, alerts, and employer proof |
| Global layout | Polished shell | Desktop underused, mobile feed not dominant enough | Rebuilt into pulse stage + three-zone desktop |
| CTA hierarchy | Safe | Not urgent enough | Sharpened primary vs secondary actions |
| Chip/tag language | Functional | Too visually similar | Split into trust, delta, salary, and state roles |
| Trust badges | Too light-touch | Needed stronger operational legibility | Increased placement density and contrast |
| Status markers | Present | Needed to feel live | Integrated into delta stage and application pulse |
